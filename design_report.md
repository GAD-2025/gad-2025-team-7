# one_day_fe_react 디자인 및 인터랙션 분석 보고서

이 문서는 `one_day_fe_react` 애플리케이션의 각 페이지 구성과 인터랙션 요소들을 설명합니다.

## 1. 전반적인 구조 (`MainLayout.js`, `ProfileHeader.js`)

대부분의 페이지는 `MainLayout` 안에 렌더링됩니다.

### 구조
- **`MainLayout.js`**:
  - `ProfileHeader`와 `Outlet` (실제 페이지 내용)을 포함하는 기본 레이아웃입니다.
  - `MainBackground` 컴포넌트가 배경을 담당합니다.
- **`ProfileHeader.js`**:
  - 상단에 표시되는 프로필 정보 헤더입니다.
  - 프로필 이미지와 닉네임을 보여줍니다.

### 인터랙션 요소
- **프로필 헤더 (`ProfileHeader.js`)**:
  - **종류**: 클릭 가능한 div
  - **기능**: 클릭 시 `/profile` 페이지로 이동하여 프로필 설정 화면을 보여줍니다.

## 2. 로그인 페이지 (`Login.js`)

### 구조
- 로그인 폼과 회원가입 폼을 포함합니다.
- 좌측에는 서비스 소개 문구가, 우측에는 폼이 위치합니다.

### 인터랙션 요소 (CTA 버튼)
- **로그인 폼**:
  - **이메일, 비밀번호 입력 필드**: 사용자의 입력을 받습니다.
  - **자동 로그인 체크박스**: 자동 로그인 옵션을 선택합니다.
  - **`로그인` 버튼 (CTA)**: `handleLogin` 함수를 호출하여 로그인을 시도합니다.
  - **`회원가입` 링크**: `showSignup` 함수를 호출하여 회원가입 폼으로 전환합니다.
- **회원가입 폼**:
  - **이메일, 비밀번호 입력 필드**: 사용자의 입력을 받습니다.
  - **이용약관, 개인정보 동의 체크박스**: 필수 동의 항목입니다.
  - **`회원가입` 버튼 (CTA)**: `handleSignup` 함수를 호출하여 회원가입을 시도합니다.
  - **`로그인` 링크**: `showLogin` 함수를 호출하여 로그인 폼으로 전환합니다.

## 3. 홈 페이지 (`Home.js`)

### 구조
- 애플리케이션의 메인 대시보드입니다.
- 상단 헤더, 캘린더, 그리고 선택된 날짜의 대시보드로 구성됩니다.
- `SlideOutNav` 컴포넌트를 통해 다른 컬렉션 페이지로 이동할 수 있습니다.

### 인터랙션 요소
- **북마크 버튼**:
  - **종류**: 버튼
  - **기능**: 클릭 시 `SlideOutNav` (슬라이드 아웃 네비게이션)를 엽니다.
- **`SlideOutNav.js` (슬라이드 아웃 네비게이션)**:
  - `DiaryCollection`, `StopwatchCollection`, `HealthcareCollection` 페이지로 이동하는 링크를 포함할 것으로 예상됩니다. (코드가 제공되지 않아 추측)
- **캘린더 (`Calendar.js`)**:
  - **날짜 클릭**: 특정 날짜를 선택하고, 해당 날짜의 정보를 대시보드에 표시합니다.
  - **드래그**: 여러 날짜를 드래그하여 기간을 선택하고, 새 일정을 추가하는 모달을 띄웁니다.
  - **이전/다음 달 버튼**: `nav` 상태를 변경하여 캘린더의 월을 변경합니다.
- **대시보드 (`Dashboard.js`)**:
  - `HomeTab`, `RecordsTab`, `HealthcareTab` 탭으로 구성됩니다.
  - 각 탭은 해당 날짜의 상세 정보를 보여주고, 데이터를 추가/수정/삭제하는 인터랙션을 제공합니다.
  - **(상세 인터랙션은 각 탭 컴포넌트에서 설명)**

## 4. 다이어리 모아보기 (`DiaryCollection.js`)

### 구조
- 사용자가 작성한 모든 다이어리를 그리드 형태로 보여줍니다.
- 상단 헤더에는 뒤로 가기, 제목, 필터 아이콘이 있습니다.

### 인터랙션 요소
- **뒤로 가기 아이콘 (`←`)**:
  - **종류**: span
  - **기능**: `handleGoBack` 함수를 호출하여 이전 페이지로 이동합니다.
- **필터 아이콘 (`📅`) (CTA)**:
  - **종류**: span
  - **기능**: `DateFilter` 컴포넌트를 표시하여 날짜 범위로 다이어리를 필터링할 수 있게 합니다.
- **`DateFilter` 컴포넌트**:
  - **`적용` 버튼**: 선택된 날짜 범위로 필터링을 적용합니다.
  - **`취소` 버튼**: 필터 창을 닫습니다.
- **필터 상태창의 `x` 버튼**:
  - **종류**: 버튼
  - **기능**: 적용된 필터를 초기화합니다.
- **다이어리 카드**:
  - **종류**: 클릭 가능한 div
  - **기능**: 클릭 시 `handleCardClick` 함수를 통해 해당 다이어리의 상세 보기 페이지 (`/diary-view/id/:id`)로 이동합니다.

## 5. 다이어리 작성/편집 페이지 (`Diary.js`)

### 구조
- 다이어리를 작성하고 편집하는 페이지입니다.
- 제목 입력 필드와 그림판(canvas)으로 구성됩니다.
- 그림판 위에는 다양한 편집 도구를 포함한 툴바가 있습니다.

### 인터랙션 요소 (CTA 버튼)
- **`저장` 버튼 (CTA)**:
  - **종류**: 버튼
  - **기능**: `saveDiary` 함수를 호출하여 현재 다이어리(그림, 텍스트, 제목)를 서버에 저장합니다.
- **툴바**:
  - **그리기 도구 (펜, 지우개, 텍스트)**: 캔버스에 그림을 그리거나 텍스트를 추가하는 모드를 선택합니다.
  - **이미지 업로드 버튼 (`🖼️`)**: 디바이스에서 이미지를 불러와 캔버스에 추가합니다.
  - **실행 취소/다시 실행 버튼 (`↩️`, `↪️`)**: 이전/다음 작업 상태로 되돌립니다.
  - **색상 팔레트**: 펜 또는 텍스트의 색상을 변경합니다.
  - **크기 조절 버튼**: 펜 또는 지우개의 크기를 변경합니다.
- **캔버스**:
  - **더블 클릭**: 텍스트 모드일 때, 캔버스에 새 텍스트 상자를 추가합니다.
  - **드래그**: 펜/지우개 모드일 때 그림을 그리거나 지웁니다.
- **텍스트/이미지 객체**:
  - **드래그**: 위치를 이동합니다.
  - **더블 클릭 (텍스트)**: 텍스트를 수정합니다.
  - **삭제 버튼 (`x`)**: 해당 객체를 삭제합니다.
  - **리사이즈 핸들 (이미지)**: 이미지 크기를 조절합니다.

## 6. 다이어리 상세 보기 (`DiaryView.js`)

### 구조
- 저장된 다이어리의 최종 결과물을 보여주는 페이지입니다.
- 상단 헤더(뒤로 가기, 제목, 날짜)와 다이어리 이미지로 구성됩니다.

### 인터랙션 요소
- **뒤로 가기 아이콘 (`←`)**:
  - **종류**: span
  - **기능**: `handleGoBack` 함수를 호출하여 이전 페이지(주로 다이어리 컬렉션)로 돌아갑니다.

## 7. 스톱워치 모아보기 (`StopwatchCollection.js`)

### 구조
- 스톱워치 기록을 카테고리별로 집계하여 막대 차트 형태로 보여줍니다.
- 상단 헤더, 정렬 필터, 날짜 필터 아이콘이 있습니다.

### 인터랙션 요소
- **뒤로 가기 아이콘 (`←`)**:
  - **종류**: span
  - **기능**: 이전 페이지로 이동합니다.
- **정렬 버튼 (`높은 순`, `낮은 순`) (CTA)**:
  - **종류**: 버튼
  - **기능**: 기록된 시간을 기준으로 카테고리를 오름차순 또는 내림차순으로 정렬합니다.
- **날짜 필터 아이콘 (`📅`) (CTA)**:
  - **종류**: span
  - **기능**: `DateFilter` 컴포넌트를 표시하여 특정 기간의 기록만 집계하도록 필터링합니다.

## 8. 헬스케어 모아보기 (`HealthcareCollection.js`)

### 구조
- 최근 7일간의 걸음 수, 소모/섭취 칼로리를 요약하여 보여줍니다.
- 각 날짜별로 데이터를 표시하고, 간단한 시각화 차트를 포함합니다.

### 인터랙션 요소
- **`← 홈으로 돌아가기` 버튼**:
  - **종류**: 버튼
  - **기능**: `/home` 페이지로 이동합니다.

## 9. 프로필 설정 (`Profile.js`)

### 구조
- 사용자 프로필 정보(닉네임, 프로필 사진)를 수정하는 페이지입니다.
- 아이디/비밀번호 변경, 로그아웃 기능도 제공합니다.

### 인터랙션 요소 (CTA 버튼)
- **`ImageUploader` 컴포넌트**:
  - **기능**: 클릭 시 파일 탐색기를 열어 프로필 사진으로 사용할 이미지를 업로드합니다.
- **`저장` 버튼 (CTA)**:
  - **종류**: 버튼
  - **기능**: `handleSave` 함수를 호출하여 변경된 닉네임과 프로필 사진을 서버에 저장합니다.
- **`뒤로` 버튼**:
  - **종류**: 버튼
  - **기능**: 이전 페이지로 이동합니다.
- **`아이디 변경하기` 버튼**:
  - **종류**: 버튼
  - **기능**: 아이디(이메일) 변경 모달을 엽니다.
- **`비밀번호 변경하기` 버튼**:
  - **종류**: 버튼
  - **기능**: 비밀번호 변경 모달을 엽니다.
- **`로그아웃` 버튼**:
  - **종류**: 버튼
  - **기능**: `handleLogout` 함수를 호출하여 로컬 저장소를 비우고 로그인 페이지로 리디렉션합니다.
- **모달 (`아이디 변경`, `비밀번호 변경`)**:
  - 각 모달은 입력을 받고, **`변경하기` 버튼 (CTA)**을 통해 해당 정보의 변경을 시도합니다.
  - **`취소` 버튼**으로 모달을 닫을 수 있습니다.
